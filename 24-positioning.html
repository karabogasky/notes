Positioning

Complex Layouts

We've seen how to use the box model and floats to create very simple web layouts, but more complex designs often require elements to be positioned.

There are four types of positioning:

  • static
  • relative
  • fixed
  • absolute

Static Positioning

Every element has a default value of static.

We do not have to specify this in our style sheet, but the property/value position: static; does exist.

It may be necessary to use this property/value to overwrite a CSS declaration for a print or mobile design.

Relative Positioning

We can position an element relative to its normal position.

To move the element, we need to specify the element's position as relative, a horizontal (x-axis) property (left or right) and a vertical (y-axis) property (top or bottom).

If we do not provide additional properties (top, right, bottom, left), the element remains in place like as if static, but has a more important state than a static element.

View Example

Fixed Positioning

We can position an element in relation to our viewport (browser window).

To move the element, we need to specify the element's position as fixed, a horizontal (x-axis) property (left or right) and a vertical (y-axis) property (top or bottom).

View Example

Absolute Positioning

An element with its position set as absolute acts like a fixed element, but relative to the nearest positioned ancestor.

If the ancestor is static, the element will be positioned in relation to the document body.

To make this work, there are two necessary steps:

  1. the containing element (ancestor) must have the property/value position: relative;
  2. a horizontal (x-axis) property (left, right), a vertical (y-axis) property (top, bottom) and position: absolute; must be specified on the element to be moved

z-index

(Or how to move elements in three-dimensional space!)

When we move elements using floats or positioning, they are taken out of the natural flow in a 3D sense as well. (Did anyone notice the green box overlapping text in the fixed positioning example?)

We can use the z-index property in CSS declarations to order positioned elements from front to back.

Higher values will place an element closer to the front.

Tip: Increment z-index values by 5 or 10 (in case you need to squeeze in another element in the future).
