Floats

Floating Elements

The float property allows us to wrap text around images and/or other elements.

The property accepts three values:

  • left
  • right
  • none

When an element is floated, it acts as a block element.

Clearing Floats

Remember how we talked about CSS 2.0 being weird and buggy? Clearing floats is a good example of how web developers had to hack code to get a design right.

There are a few solutions, all of which may work in different situations.

Try what works in your design. If it does, stick with it.

clear property

Sometimes we need content within the normal flow to appear after a floated element, as if the content were starting on a new line or a beginning a section.

To end a float and re-establish the normal flow, we use the clear property.

If we're trying to move past:

  • left-floated elements, use clear: left;
  • right-floated elements, use clear: right;
  • left- and/or right-floated elements, use clear: both;

The clear property also takes the value none.

:after pseudo-class

If the floated elements are contained (inside a parent element), apply the following declarations using the :after pseudo-class on the parent element.

For example:

section:after {
  content: "";
  display: block;
  clear: both;
}

.clearfix class

Another popular way of clearing a float is to enclose the floated element within a container and add a clearfix class to the parent (the container) of the floated element in HTML.

.clearfix {
  overflow: auto;
  zoom: 1;
}

Because clearfix is a class, it can be used multiple times in our HTML file.

This is a pretty standard "hack" which still works in older browsers.

It's not the nicest solution, but sometimes you go with what works.
