Print Stylesheets

Think about media

When we consider different media, there are differences which set each apart from the other.

When you compare web to print, print to mobile, mobile to web, what are some of those minor differences?

Styling for print

Since the web can be transformed to printed media, we can style for it.

Basics

Like the web, we never know how a user will consume our site. The same goes for how a user may print our site.

What kind of printer do they have? Color? Black and white? Laser? Inkjet? Dot matrix?

Always aim for the lowest-common-denominator: black and white.

Add a link to a print stylesheet

<link rel="stylesheet" media="print" href="print.css" />

Make sure you set media to print. Otherwise, your print stylesheet will not work!

Set your foundation

* {
  color: black;
  background: transparent;
  font-size: Npt;
  font-family: serif;
}

color: black;

This will eliminate any color specified on your stylesheet.

background: transparent;

This will eliminate any background images, background colors which may waste a significant amount of ink.

font-size: Npt

Set your font size to whatever you please (hence the value 'N'), just make sure you are using print units, not screen units.

Example: font-size: 24pt;

Be aware!

When you reset the font-size using the universal selector, remember, it will overwrite every font size, including browser defaults. So if your h1-h6 tags are helping establish a visual hierarchy, you will need to set separate font sizes for those as well.

font-family: serif;

From a design standpoint, sans-serif fonts generally provide a better experience for reading on-screen text. Serif fonts are always better for print media.

@page

@page provides styles for the page we're printing on, so this is where we establish margins.

Again, use print units, not screen units.

Example: @page { margin: 0.5in; }

Print units

A reminder about print units. Remember, you want to use:

  • points
  • inches
  • centimeters
  • millimeters

When do I use unit X?

When the item you're styling will be displayed -- borders, fonts, for instance -- use points.

Example: section { border: 1pt solid black; }

When the item you're styling is invisible -- margin, padding -- use inches, centimeters or millimeters. The choice is yours. Just be consistent in how these are applied in your design.

Example: @page { margin: 1in; }

Let's think about links

Links are unique to the web. Do we need links in print?

It depends.

When we provide a link inside text, it is the equivalent providing a source. Think of it as the digital equivalent of a footnote.

How can we add those sources in printed media?

Before we decide

Underlines always indicate links on the web. In print, underlines may convey something else. So we should probably remove those.

a { text-decoration: none; }

Displaying Links in Print

By combining attribute selectors which search for external links (which always begin with http:// or https://), and using the content property, we can insert the content of our href attribute after the text.

This gives a user with a printed page the ability to find the web-based work we are referencing.

a[href^="http://"] {
  display: inline;
}

a[href^="http://"]:after {
  content: " (" attr(href) ") ";
}

What about internal links?

Ideally, if we have created print style sheets for our entire site, a user can visit another page and print that content just as easily.

We could set our internal links (which more often than not will end in .html) to display: none;.

Example: a[href$=".html"] { display: none; }

But again, we never know how our content will be consumed. If it's necessary to reference links to internal pages, we could construct a rule as such (rather than setting the display property to none):

a[href$=".html"]:after {
  content: " (http://domain.com/" attr(href) ")"; }
}
