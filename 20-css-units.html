CSS: Units of Measurement

Choices, choices...

What units of measurement do we use when specifying box model, width, height, etc. properties?

We utilize screen units, including:

  • pixels (px)
  • percentage (%)
  • ems (em)
  • root ems (rem)

The Web is Not Print

When you design for print, you may use the following units of measurement:

  • points
  • picas
  • inches
  • centimeters
  • millimeters

DO NOT use print units when designing for digital media. Use screen units.

One exception: Print style sheets.

Pixels

Pros

  • allows for very precise control of design
  • what you see is what you get (WYSIWYG)

Cons

  • poor choice for fluid, responsive design
  • not ideal for resizing text, designs

Syntax

font-size: 30px;

Relative Units

Percentage, ems and rems are relative units.

These values are based on a pixel measurement set within the browser's default preferences or at the html or body level.

Elements will then adjust their measurement relative to that initial value.

Percentage

Pros

  • Most HTML default values are percentage based
    • h1 is 200% of the default font size
    • block elements take up 100% of the parent element's width
  • great for fluid, responsive design

Cons

  • requires calculation based on parent font size
  • very precise calculations can disrupt layout

Syntax

width: 90%;

Ems

Ems are a typographic measurement which reference the width of the capital "M" in a typeface being used.

It is not in any way related to <em>.

Pros

  • great for fluid, responsive design
  • once a base font is set, everything can be relative (to the parent)

Cons

  • requires calculation based on parent font size
  • very precise calculations can disrupt layout
  • inherits a parent's property

Syntax

margin: 2em; /* 32px, if base font is 16px */

Root Ems

Root ems (rem) were introduced in CSS3.

They are like ems, but relative to the root (html) element, not a parent element.

Pros

  • a little more predictable than percentage or ems
  • great for fluid, responsive design

Cons

  • requires calculation
  • can still result in precise, layout-disrupting calculations
  • older browsers (IE 8 or less, Opera Mini) lack support

Syntax

font-size: 1.25rem; /* 20px if base font is 16px */
