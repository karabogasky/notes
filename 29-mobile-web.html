Designing for the Mobile Web

Think about media (again)

How does a web site differ from a mobile site?

Visit some of your favorite sites. Note the differences. What is similar? What is different?

Web & Mobile

Some web sites actually host two separate sites for the web and mobile. There are negatives to this:

  • more infrastructure
  • repeated efforts
  • redirection (which can impact SEO)

Examples

  • New York Times
  • New York Times (Mobile)

There has to be a better way, right?

Benefits of Responsive Design

By creating a responsive, fluid design, we can use one HTML file and adjust the styling based on the device being used to browse the site.

Example

  • Boston Globe
  • Fox Sports Florida

Things to consider

  • screen size
  • bandwidth
  • browser capabilities
  • device capabilities

@media queries

@media queries allow us to specify styles based on conditions.

@media (max-width: 320px) { }

...will only apply styles if the maximum width of the window is 320 pixels.

Media features

  • aspect-ratio
  • device-aspect-ratio
  • device-height
  • device-width
  • height
  • orientation
  • resolution
  • width

And some features not used as frequently:

  • color
  • color-index
  • grid
  • monochrome
  • scan

Two components for the mobile web

  1. meta tag specifying viewport
  2. a mobile stylesheet with an @media query

HTML

<meta name="viewport" content="width=device-width; initial-scale=1;" />

meta viewport

Adding meta viewport was a solution put forth by Apple. Many browser and mobile device makers adopted it. But not Microsoft or more importantly, the W3C. The W3C is working on a standard which would have us apply this setting through CSS.

The @viewport rule is only supported by Opera and IE 10 and requires vendor prefixing.

Vendor Prefixing

Just an aside here...

As new CSS features are added to browsers, they're released in a beta state — they might work, but maybe not according to W3C specs.

So certain CSS properties are 'prefixed' with browser identification:

  • -ms-: Microsoft
  • -moz-: Mozilla (Firefox)
  • -o-: Opera
  • -webkit-: Safari, Chrome (webkit browsers)

Maximum Scale

If you read mobile web tutorials, some may suggest using an attribute/value pair: maximum-scale=1; in meta viewport.

DO NOT include this attribute.

It hinders the ability for a user to pinch zoom your web site, making it less accessible.

More HTML

You also need to add a link to your mobile stylesheet AFTER you link to your (web) screen.css.

<link rel="stylesheet" media="screen" href="css/mobile.css" />

Why? Because the last styles written 'win out.'

The catch

There's a @media property: handheld.

But today's mobile browsers (iOS, Android) don't support it. So don't use it.

Apple and Google touted their devices as being able to view the web — as it is — and disabled the use of handheld.

So... we have to use screen with set widths (and/or orientation) and overwrite rules created for the web.

Setting up your CSS

It's a good idea to keep your screen and mobile styles separate.

Start with the following in a new document:

@media (max-width: 320px) {
  ...
}

Modifying your site for mobile

Mobile design basically involves overwriting screen rules you created for the web version of your site.

Copy each of the rules from your web style sheet (screen.css) into your mobile stylesheet and adjust them line-by-line.

If you do not overwrite rules from screen.css, the mobile version of the site will inherit those rules.

Suggestion: Don't start building for mobile until you've finalized your design for web.

Some questions to ask

Should the font size be bigger? (Yes.)

Do you have to remove floats or positioning?

Are there elements which might be 'hidden' in a mobile design? (Yes. Multimedia, perhaps.)

Do I need to adjust HTML to include content to only be displayed on mobile?

Different designs will pose different challenges. Be sure to give your site a critical eye.

Beyond smartphones

Mobile design doesn't have to be limited to tiny screens.

You can create styles for different breakpoints — -- different max-width values based on screen sizes.

A great, fluid design may not need breakpoints. But the more complex a design (i.e. Boston Globe) the more necessary they become.
