Why so many browsers?

Freedom of choice! :)

Forced to adopt web standards, browser makers discovered the browser itself could be a commodity by adding features and services.

Remember: Open-source design encourages participation, experimentation.

Browser makers try to improve, expand our browsing experience. (It also reinforces their corporate brand.)

Understanding Browsers

  • Browser has a layout and rendering 'engine'
    • Many based on open-source code
    • Makers aim for improved speed, user experience
  • Modern browsers follow W3C standards
  • New standards are implemented as they are approved by W3C...
  • how quickly a feature is added depends on the browser maker

What is a server?

Computers which are set up solely to handle service requests from clients. Users typically connect to a server through a network (i.e. the Internet).

  • Database
  • File
  • Email
  • Print
  • Web

A server can be a single, physical machine which handles multiple tasks. For speed and efficiency, sometimes a server is dedicated to handling a single task.

Application Protocols

Computers and servers connect through transfer protocols (i.e. TCP/IP).

Tools which interact with network have their own application protocols.

  • FTP: File Transfer Protocol
  • NNTP: Network News Transfer Protocol
  • SSH: Secure Shell
  • SMTP: Simple Mail Transfer Protocol

All require TCP/IP in order to work.

HTTP

The web has its own protocol: Hypertext Transfer Protocol.

  • facilitates communication between a client and server
  • a client (web browser) requests a file from a web server
  • the web server acknowledges the request by sending response
  • if the response is "OK!", the requested file is sent

Identifying Sites & Pages

How do we indicate which web site or file we are looking for on a server?

HTTP relies on a client providing a URL, or Uniform Resource Locator, a standard way of identifying a server, directory or file on the web.

A URL's structure consists of:

  • scheme name (protocol used to transfer the file)
    • http:
    • https:
  • hierarchy (the path to the file)
    • //com.miami.edu/about/about-the-school

Requests

The most common requests a client can make to a server are GET and POST.

  • GET retrieves data
    • most common request when browsing the web
  • POST sends data back to a server to be saved
    • post to a message board, update a database, submit a form

HTTP Status Codes

When a client makes a request, it will respond with a status code.

  • 1xx: Informational
  • 2xx: Success
    • 200: OK! (file transfer begins)
  • 3xx: Redirection
    • 301: Moved Permanently
  • 4xx: Client Error
    • 404: File Not Found
  • 5xx: Server Error
    • 500: Internal Server Error
