span & div

Generic Elements

Up to this point, we've talked about HTML tags having meaning.

Whenever you can use a tag with meaning, you should.

Sometimes, though, we need to use elements for the sole purpose of implementing a style within a design. For this we use span and div.

span

  • a generic inline container
  • can be used with id and class attributes
  • should only be used when no semantic element is appropriate

Examples of span

Bad

<p>We code in <span>HTML</span></p>

Why? Because we could use abbr instead.

Good

<p>This is <span class="red">red text</span></p>

div

  • a generic block container
  • can be used with id and class attributes
  • should only be used when no semantic element is appropriate

Examples of div

Bad

<div>This is a sentence in a paragraph.</div>

Why? We should be using a p instead.

Good

<div class="container">
  <header>
    <h1>Erin's Web Site</h1>
  </header>

  <main>
    <p>Hello! Welcome to my page</p>
  </main>
</div>

Reminder

Always aim to use a proper semantic tag before resorting to use of a span or div.

Only use span and div after creating a well-structured document, when it is clear additional elements are needed for styling purposes.
