CSS Selectors

Universal Selector

* { color: red; }

  • Applies to all elements in a document
  • In this case, we are changing the color of all the content (text) to red

View Example

Type Selector

p { color: red; }

  • Matches elements
  • We're changing all content within <p> to red

We can also group elements into one CSS block:

h1, h2, h3, h4, h5, h6 { color: red; }

View Example

Class Selector

.red-text { color: red; }

p.red-text { font-family: sans-serif; }

  • Matches any element with the specific class value
  • It can also be applied to an element with the class value attached
  • In the second example, only <p class="red-text"> will be impacted

ID Selector

#flynn { border-width: 1px; }

  • Matches any element with the specific id value
  • It can also be applied to an element with the id value attached, but...
    • Theoretically, this is unnecessary since id is unique

Child Selector

li > a { text-decoration: none; }

  • Matches an element which is a child of the preceding element
  • In this case, if <a> is a child of <li> the default underline applied to links is removed

View Example

Descendant Selector

section a { text-decoration: none; }

  • Matches an element which is a descendent of the parent element
  • It does not have to be a direct child
  • In this case, we're removing the default underline applied to links from any <a> element nested within a <section> (regardless of whether the <a> is nested in a <article>, <p>, etc.)

View Example

Adjacent Sibling Selector

h1 + p { font-weight: bold; }

  • Matches the element which is next to its sibling
  • In this example, we find <h1> and change its first sibling, <p> to bold.
  • No other <p> elements will be impacted by this declaration

View Example

General Sibling Selector

h1 ~ p { color: red; }

  • Matches an element which is a sibling
  • The sibling does not need to be directly following
  • So, if you had siblings <h1>, <h2>, <p> and <p>, the text of the two <p> elements would be red.
