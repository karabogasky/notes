Document Object Model

Document Object Model (DOM)

  • A tree-like structure which helps us interact with HTML
  • Document: Our .html file
  • Object: HTML elements (also called nodes)
  • Model: The structure we create using elements
    • nodes are nested
    • nodes are described parents, children and/or siblings

    • "Welcome to my web site."

The HTML family tree

<html>

  • has no parents
  • head and body are its children
  • head is its first child
  • body is its last child

<head>

  • body is its sibling
  • title is its child
    • the "text node" (or content) is "My Web Site"

<body>

  • has one sibling and two children
  • head is the sibling
  • h1 and p are its children
    • h1 is the first child
    • p is the last child

<h1> and <p>

  • are siblings
  • h1 has one text node
    • "Hello!"
  • p has one text node
    • "Welcome to my web site."

A quick lesson

In computer science parlance, an array (collection) of items will always start with the number zero (0).

array = [apple, banana, carrot]

  • array[0] is "apple"
  • array[1] is "banana"
  • array[2] is "carrot"

This concept has been carried over to the DOM.

When there are two nodes with the same name, we can specify each individually by indicating the order in which they are in (0, 1, 2, n...).

The HTML family tree (revisited, again)

We can identify our tr siblings as follows:

  • tr[0] is the first table row
  • tr[1] is the second — in this case, last — table row

Remember, this is just how we describe the path. We never type the [n] in HTML. We (may but) rarely use it in CSS and/or Javascript.

  • Each tr has two children
    • tr[0] has two children
      • th[0]
      • th[1]
    • tr[1] also has two children
      • td[0]
      • td[1]

Paths

  • table » tr[0] » th[0] » "City"
  • table » tr[0] » th[1] » "State"
  • table » tr[1] » th[0] » "Coral Gables"
  • table » tr[1] » th[1] » "Florida"

In plain words

table » tr[0] » th[0] » "City"

  • "table is the parent element."
  • "tr[0] is the first child of table."
  • "th[0] is the first child of tr[0]."
  • "'City' is the text node."
  • "Find the content in table, the first tr and the first th."
