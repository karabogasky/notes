Dimensions & Overflow

Sizing Boxes

As we have seen with the block elements and the box model, content will take up whatever room we give it... unless we specify otherwise.

When we want to create layouts, we can apply constraints:

  • width
  • height
  • overflow

Dimensions

Width is often helpful for layout and design since we are often limited (horizontally) by screen size.

We can specify height but it really is not necessary since vertical scrolling is an accepted practice when using a web interface.

  • width
  • max-width
  • min-width
  • height
  • max-height
  • min-height

Overflow

What happens if we have content which is greater in size than the box we've provided for it?

We can use the overflow property, which handles how that content is displayed within its parent element.

  • visible
  • hidden
  • scroll
  • auto

visible

The visible value breaks the box's dimensions.

hidden

The hidden value hides the content which would normally extend beyond the box's dimensions.

scroll

The scroll value adds vertical and horizontal scroll bars to the containing box.

If the content is flexible (horizontally) within containing box, only vertical scroll bars will be added.

auto

The auto adds vertical and/or horizontal scroll bars if necessary.

A new discovery?

As you saw from the examples, it seems scroll and auto work similarly: The browser will determine whether scrollbars are needed.

Keep in mind different browsers, operating systems may treat these differently than what you see here. Understand how the spec works so you make the right choice for your design and browser compatibility.
